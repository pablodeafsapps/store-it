Regarding section 1.1 from #file:CONVENTIONS.md , I'd like you to document to communicate modules, so that there are no circular dependencies and structure remains clear and legible during time.

This strategy is based on "gateways", which is fundamentally based on the Strategy design pattern and the DI engine (Koin in our case).

The idea is that every feature module which requires data/functionality from another feature module must implement an interface, with some functions of interest, which is declared in the :core module. Doing so, this interface gets accessible by any other module depending on :core, which happens by default. The DI engine is in charge to resolve this interface to the actual implementation defined in the feature module which owns the data/functionality.
